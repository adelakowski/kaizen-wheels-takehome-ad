  [x] page.tsx passes searchParams → SearchPage → VehicleListLoader
  [x] Suspense key on filter string re-fetches fleet on Show cars / URL change
  [x] Removed client-side app/lib/filters.ts (all filtering in Postgres)

Filter rationale (PRD §3.2)
  - Dates + availability: hard constraint — unavailable cars never shown when dates set
  - Class: highest-signal body-type filter after dates (trip purpose)
  - Daily price: budget anchoring before emotional investment in a car
  - Passengers: hard fit constraint for families/groups
  - Make: brand loyalty / familiarity entry point

UX (PRD §3.4)
  [x] No dates → full fleet shown; lime prompt to select dates + Show cars
  [x] Dates set → unavailable vehicles excluded; count says "vehicles available"
  [x] No dates → count says "vehicles in fleet"; Select disabled until dates set (VehicleCard)
  [x] Contextual empty states (no availability vs no filter matches)
  [x] Review getQuote rejects CONFLICT if vehicle booked for window
